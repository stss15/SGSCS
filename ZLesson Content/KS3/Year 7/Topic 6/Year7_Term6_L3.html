────────────────────────────

A Quick Recap


Before we dive into adding binary numbers, let’s review a few key points:

 • Binary Basics: Computers use binary because each tiny electronic switch can only be either on (1) or off (0). Every piece of digital data—from images to games—is built from these bits.

 • Place Values in Binary: In an 8-bit binary number, each bit has a value based on a power of 2. The rightmost bit represents 2⁰ (or 1), and each bit to the left has double the value of the one before it; for example, the leftmost (8th) bit represents 2⁷ (or 128).

 • Hexadecimal Shortcut: In Lesson 2, we learned that hexadecimal is a base‑16 system that condenses long binary strings into shorter symbols—each hex digit equals four binary digits.


With these ideas fresh in your mind, you’re ready to see how computers actually perform calculations using binary addition.


────────────────────────────

Adding Two 8-Bit Binary Numbers


The Basic Idea

Adding binary numbers is much like the decimal addition you do every day, except you only work with two digits: 0 and 1. When you add two binary digits:

 • 0 + 0 = 0  (no carry)

 • 0 + 1 = 1  (no carry)

 • 1 + 0 = 1  (no carry)

 • 1 + 1 = 10 in binary  (write 0 and carry 1 to the next column)


These simple “recipes” allow you to build up larger numbers from bits.


────────────────────────────

Step-by-Step Example


Let’s add two 8-bit binary numbers. Consider these numbers:


  Number A: 01010110   (which is 86 in decimal)

  Number B: 00111011   (which is 59 in decimal)


We will add these numbers column by column (from right to left). Write them in column format:


     01010110

  + 00111011

  ───────────


Now, let’s add each column with care:

1. Column 8 (1s place):

  A: 0  B: 1

  0 + 1 = 1  → Write 1; no carry.

2. Column 7 (2s place):

  A: 1  B: 1

  1 + 1 = 10 in binary.

  → Write 0; carry 1 to the next column.

3. Column 6 (4s place):

  A: 1  B: 0; plus the carried 1 gives: 1 + 0 + 1 = 10.

  → Write 0; carry 1.

4. Column 5 (8s place):

  A: 0  B: 1; plus the carried 1 gives: 0 + 1 + 1 = 10.

  → Write 0; carry 1.

5. Column 4 (16s place):

  A: 1  B: 1; plus the carried 1 gives: 1 + 1 + 1 = 11 in binary.

  → Write 1; carry 1.

6. Column 3 (32s place):

  A: 0  B: 1; plus the carried 1 gives: 0 + 1 + 1 = 10.

  → Write 0; carry 1.

7. Column 2 (64s place):

  A: 1  B: 0; plus the carried 1 gives: 1 + 0 + 1 = 10.

  → Write 0; carry 1.

8. Column 1 (128s place):

  A: 0  B: 0; plus the carried 1 gives: 0 + 0 + 1 = 1.

  → Write 1; no further carry.


When you combine the digits from left (Column 1) to right (Column 8), the final result is:


  Result: 10010001


To double-check, convert 10010001 to decimal:

  1×128 + 0×64 + 0×32 + 1×16 + 0×8 + 0×4 + 0×2 + 1×1 = 128 + 16 + 1 = 145

This matches the sum of 86 and 59.

Understanding Overflow


Overflow occurs when the result of an addition exceeds the maximum value that can be stored in 8 bits (which is 255, since 11111111 in binary equals 255).


Imagine you have a small cup that can hold only 255 millilitres of water. If you try to pour in 300 millilitres, the extra water spills out. Similarly, if you add two 8-bit numbers and the sum is greater than 255, the extra value “overflows” and is lost. This can lead to errors in calculations.


For example, consider adding two larger numbers:

  • 200 in 8-bit binary: 11001000

  • 100 in 8-bit binary: 01100100

Their true sum is 300, but since 300 exceeds 255, only the last 8 bits are stored, and the overflow is discarded. This might lead to unexpected results in a program or device.


────────────────────────────

Real-World Examples of Binary Addition and Overflow


Digital Clocks:

Many digital clocks use binary arithmetic to count seconds. If an 8-bit counter reaches its maximum value (255) and overflows, the clock must either reset or use additional bits to continue counting correctly. Overflow management ensures the clock displays the correct time.


Video Games:

In video games, binary arithmetic is used for scoring, tracking lives, or managing in-game timers. An overflow in an 8-bit system might cause a score to drop suddenly or display an error, which is why game designers often use larger bit systems to avoid such issues.


Communication Devices:

When sending data over networks, devices perform numerous binary calculations. Overflow errors in these operations could result in data corruption or transmission glitches. Reliable systems are designed to detect and handle overflow, ensuring data integrity.

Recap and Key Points


Let’s summarise what we’ve learned in this lesson:

 1. Adding 8-Bit Binary Numbers:

  - Write the numbers in column format and add from right to left.

  - When the sum of a column reaches 2 (or more), write down the appropriate digit and carry the extra value to the next column.

  - In our example, adding 01010110 (86) and 00111011 (59) gave us 10010001 (145).


 2. Understanding Overflow:

  - An 8-bit system can only represent values from 0 to 255.

  - If an addition produces a number greater than 255, the extra bits “overflow” (like water spilling from a cup), which can cause errors.


 3. Real-World Applications:

  - Digital clocks, video games, and communication devices all rely on precise binary arithmetic.

  - Effective handling of overflow is crucial to ensure these systems work correctly.
